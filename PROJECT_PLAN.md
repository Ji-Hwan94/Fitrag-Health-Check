# LLM 기반 크로스핏 맞춤 운동 및 식단 추천 서비스 기획서

## 1. 프로젝트 개요

### 프로젝트명
FitRAG Coach

### 프로젝트 목적
사용자가 현재 신체 정보와 목표 신체 정보를 입력하면, LLM과 RAG를 활용해 크로스핏 중심의 주간 운동 계획과 식단 계획을 추천하는 개인 맞춤형 건강 관리 서비스이다. 사용자는 목표 체중, 근육량, 체지방량 등을 기준으로 현실적인 운동량과 영양 섭취 방향을 확인하고, 식단에 필요한 식재료를 쿠팡 등 쇼핑 사이트에서 바로 탐색할 수 있다.

### 핵심 가치
- 개인 신체 정보 기반 맞춤형 크로스핏 운동 계획 제공
- 목표 체중, 근육량, 체지방량에 따른 식단 및 영양 전략 추천
- RAG를 활용한 신뢰 가능한 운동/영양 지식 기반 답변
- 식단 실행을 돕는 식재료 구매 연동
- 초보자도 이해하기 쉬운 주간 실천 계획 제공

## 2. 문제 정의

많은 사용자가 체중 감량, 근육 증가, 체지방 감소를 목표로 운동과 식단을 시작하지만 다음과 같은 어려움을 겪는다.

- 본인의 신체 상태에 맞는 운동 강도와 빈도를 알기 어렵다.
- 크로스핏 운동은 종류와 강도가 다양해 초보자가 계획을 세우기 어렵다.
- 식단 정보는 많지만 목표에 맞는 섭취량과 식재료 구성이 어렵다.
- 추천받은 식단을 실제 장보기로 연결하는 과정이 번거롭다.
- 인터넷 정보는 출처가 불분명하거나 개인 상태에 맞지 않는 경우가 많다.

이 프로젝트는 LLM 기반 대화형 추천과 RAG 기반 지식 검색을 결합해, 사용자가 본인에게 맞는 운동/식단 계획을 빠르게 이해하고 실행할 수 있도록 돕는다.

## 5. 핵심 기능

### 5.1 신체 정보 기반 목표 분석

사용자가 입력한 신체 정보를 바탕으로 현재 상태와 목표 상태의 차이를 분석한다.

주요 결과:
- 현재 BMI 및 체중 상태
- 목표 체중까지 필요한 감량/증량 폭
- 목표 근육량 달성을 위한 예상 기간
- 건강한 변화 속도 안내
- 부상 또는 과훈련 위험 경고

### 5.2 크로스핏 기반 주간 운동 추천

사용자의 운동 경험, 목표, 주당 가능 횟수에 따라 크로스핏 중심의 운동 루틴을 추천한다.

추천 예시:
- 주 3회 초보자용 크로스핏 루틴
- 주 4~5회 체지방 감량 중심 루틴
- 근육량 증가를 위한 근력 보강 포함 루틴
- WOD, 스킬 훈련, 근력 훈련, 회복 운동 구성

운동 구성 요소:
- 워밍업
- 기술 연습
- 근력 운동
- 메인 WOD
- 쿨다운
- 회복/스트레칭

### 5.3 목표 기반 식단 추천

운동 목표에 맞춰 일일 칼로리와 영양소 비율을 산정하고 식단을 추천한다.

추천 내용:
- 하루 권장 칼로리
- 단백질/탄수화물/지방 목표량
- 아침/점심/저녁/간식 식단
- 운동 전후 식사 가이드
- 감량기/증량기/유지기 식단 전략
- 외식 시 대체 메뉴

### 5.4 식재료 쇼핑 연동

추천 식단에 필요한 식재료를 쇼핑 사이트에서 확인할 수 있도록 연결한다.

초기 구현 방식:
- 식단별 필요한 식재료 리스트 생성
- 식재료별 검색 키워드 생성
- 쿠팡 검색 URL 연결
- 향후 파트너 API 또는 커머스 API 연동

예시:
- 닭가슴살
- 현미밥
- 그릭요거트
- 계란
- 고구마
- 냉동 블루베리
- 단백질 보충제

### 5.5 RAG 기반 신뢰도 강화

운동과 영양 관련 답변에 RAG를 필수 적용한다. LLM이 단순 추측으로 답하지 않고, 사전에 구축한 신뢰 가능한 문서 데이터베이스를 검색한 뒤 답변을 생성하도록 한다.

RAG 활용 영역:
- 운동 강도 추천 근거
- 단백질 섭취량 기준
- 감량/증량 속도 기준
- 크로스핏 운동 설명
- 부상 예방 가이드
- 회복과 수면 가이드
- 식단 제한 조건별 대체 식품

## 6. RAG 설계

### 6.1 지식 데이터 소스

초기 데이터 후보:
- ACSM 운동 가이드라인
- WHO 신체활동 가이드라인
- ISSN 스포츠 영양 포지션 스탠드
- CrossFit 공식 운동 설명 문서
- 식품영양성분 데이터
- 국내 식품의약품안전처 식품영양성분 DB
- 자체 작성 운동/식단 가이드 문서

주의:
의료 진단이나 치료 목적의 답변은 제공하지 않으며, 질환/부상/임신/섭식장애 등 고위험 조건이 있는 사용자는 전문가 상담을 권장한다.

### 6.2 문서 처리 방식

1. 문서 수집
2. 텍스트 추출
3. 문서 단위 정제
4. 청크 분할
5. 임베딩 생성
6. 벡터 DB 저장
7. 사용자 질문 또는 추천 요청 발생
8. 관련 문서 검색
9. LLM 답변 생성
10. 답변에 참고 근거 제공

### 6.3 벡터 DB 후보

MVP:
- Chroma
- FAISS

서비스 확장:
- Pinecone
- Weaviate
- Qdrant
- PostgreSQL + pgvector

### 6.4 RAG 답변 정책

- 검색된 문서 근거가 부족하면 확정적으로 답하지 않는다.
- 사용자의 건강 상태에 위험 요소가 있으면 보수적으로 안내한다.
- 운동 강도는 사용자의 경험 수준과 회복 가능성을 반영한다.
- 식단은 극단적인 칼로리 제한을 피한다.
- 출처 또는 근거 문서를 함께 표시한다.

## 7. 사용자 시나리오

### 시나리오 1: 체중 감량 사용자

1. 사용자가 키 175cm, 체중 82kg, 목표 체중 74kg을 입력한다.
2. 주 3회, 하루 60분 운동 가능하다고 입력한다.
3. 서비스가 감량 목표와 예상 기간을 분석한다.
4. 주 3회 크로스핏 루틴과 주 2회 가벼운 유산소/회복 운동을 추천한다.
5. 하루 권장 칼로리와 고단백 식단을 제안한다.
6. 필요한 식재료를 쿠팡 검색 링크로 제공한다.

### 시나리오 2: 근육량 증가 사용자

1. 사용자가 현재 체중 68kg, 목표 체중 73kg, 목표 근육량 증가를 입력한다.
2. 운동 경험은 중급, 주 5회 운동 가능하다고 입력한다.
3. 서비스가 근력 훈련 비중이 높은 크로스핏 루틴을 추천한다.
4. 단백질과 탄수화물 섭취량을 늘린 식단을 제안한다.
5. 운동 후 회복 식사와 수면 가이드를 제공한다.

## 12. 추천 로직 초안

### 운동 추천 기준
- 초보자: 주 2~3회, 낮은 강도, 기초 동작 중심
- 중급자: 주 3~5회, WOD와 근력 훈련 병행
- 고급자: 주 5회 이상, 목표별 고강도 구성

### 식단 추천 기준
- 감량 목표: 유지 칼로리 대비 완만한 칼로리 적자
- 증량 목표: 유지 칼로리 대비 완만한 칼로리 흑자
- 근육 증가: 충분한 단백질과 운동 전후 탄수화물 강조
- 체지방 감소: 단백질 유지, 가공식품과 액상 칼로리 제한

## 13. LLM 프롬프트 방향

### 시스템 프롬프트 핵심
- 너는 운동 및 영양 계획을 돕는 코치이다.
- 의료 진단을 하지 않는다.
- 사용자의 입력값과 검색된 RAG 문서를 근거로 답한다.
- 불확실한 경우 전문가 상담을 권장한다.
- 초보자가 이해할 수 있는 언어로 설명한다.
- 무리한 감량, 과훈련, 극단적 식단을 추천하지 않는다.

### 출력 형식
- 현재 상태 요약
- 목표 분석
- 주간 운동 계획
- 식단 계획
- 식재료 리스트
- 주의 사항
- 참고 근거

## 19. 결론

이 프로젝트는 LLM의 개인화 추천 능력과 RAG의 신뢰도 보강을 결합해, 사용자가 목표 체중과 근육량을 향해 실행 가능한 운동/식단 계획을 세울 수 있도록 돕는 서비스이다. 특히 크로스핏이라는 명확한 운동 도메인을 중심으로 하고, 식단 추천을 실제 식재료 구매까지 연결한다는 점에서 일반적인 건강 챗봇보다 실행력이 높은 서비스로 발전할 수 있다.
