# MVP 개발 로드맵

FitRAG Coach를 4주 안에 검증 가능한 MVP로 구현하기 위한 개발 계획이다.

## 1. MVP 목표

사용자가 신체 정보와 목표를 입력하면 신체 분석, 주간 운동 계획, 하루 식단 예시, 식재료 검색 링크, RAG 근거를 한 번에 받을 수 있는 흐름을 완성한다.

## 2. MVP 성공 기준

- 사용자가 첫 추천 결과를 받을 수 있다.
- BMI, 목표 변화량, 기본 칼로리와 영양소 목표가 계산된다.
- 주간 크로스핏 운동 계획이 생성된다.
- 하루 식단 예시와 식재료 리스트가 생성된다.
- 최소한의 RAG 근거가 추천 결과에 표시된다.
- 건강 위험 조건에 대한 주의 문구가 표시된다.

## 4. 우선순위

### P0 필수

- 신체 정보 입력
- 목표 입력
- 신체 분석
- 운동 계획 생성
- 식단 계획 생성
- RAG 근거 표시

### P1 중요

- 추천 결과 저장
- 장보기 검색 링크
- 위험 조건 감지
- 대체 운동과 대체 식품

### P2 후순위

- 로그인 고도화
- 즐겨찾기
- 가격 추정
- 추천 결과 공유
- 사용자 피드백 수집

## 5. 기술 선택안

### 빠른 MVP 조합

- Frontend: Next.js, Tailwind CSS, Zustand
- Backend: FastAPI
- Database: PostgreSQL 또는 Supabase
- Vector Store: Chroma
- LLM/RAG: OpenAI API, LangChain 또는 LlamaIndex
- Deployment: Vercel, Render 또는 [Fly.io](http://Fly.io)

## 6. 리스크와 대응

### RAG 문서 준비 지연

- MVP에서는 자체 작성 가이드 문서와 공개 가이드라인 요약부터 사용한다.

### 추천 품질 편차

- 프롬프트 출력 형식을 고정하고, 위험 조건 감지 규칙은 코드 레벨에서 별도로 처리한다.

### 건강 정보 책임 이슈

- 의료 진단이 아님을 명확히 표시하고, 고위험 사용자는 전문가 상담을 권장한다.

### 쇼핑몰 정책 변경

- 초기에는 검색 URL 생성만 제공하고, 공식 API 연동은 MVP 이후 검토한다.
